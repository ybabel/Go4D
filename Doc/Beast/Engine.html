ALOM3D : 3D rendering engine

What can be done with it :

  • 6 degree of freedom
  • Gouraud : a normal (average of the normal of the faces that use a plot) must be processed for each plot
  • Bump : a dynamic light map must be processed
  • Draw faces, points, lines. A single face can be drawed by various maner, depending on the speed required (linear mapping for fast rendering, hypperbolic+mipmap+alphablending+ZBuffer+phong lightmap+transluency+fog for accurate rendering)

  • easily inculde of Physic rule, collision detection, AI
  • Hierarchical objects definition
  • Hierarchical bounding box or sphere of fast collision detection
  • Polygon accuray collision detection
  • Render the cube demo
  • Render the IIE demo
  • Render the physic demo
  • Be integrated in the interactive 3D editor/modeler
  • Used to render outdoor/indoor scenes
  • Render the beast demo (without much time consuming in rendering)
  • Read vrml, asc or other objects definition files
  • Include voxel engine !!! to draw the environment
  • Possibility to include CSG later

  • BSP ?!?
  • Dynamic moving lights, ambient lights, spot lights, colored lights (like the java3D sphere lighted with green and red)

Rendering paradigm :

  • The Space system
  • The View system
  • The Camera (make link from space to view, transform space into view)
  • The projector (that project the space objets into a view object : X,Y,Z+RGB -> X,Y+Color) = camera
  • The rasterizer (previously known as the TemplateFaceDrawer)

The rendering pipeline :

  1. The objects high level definitions, the only objects manipulated by the game/demo : include objects (cube+sphere+face defined objects+composite objects) + lights + physics + constraints + dynamic. The objects must be said Static/dynamic in order that the engine knows if it can optimze their use or not. Objects can be defined by hand, by a program (beasts), or in a file
  2. Objects position computing into camera base
  3. High level visibility determination (use bounding sphere and frustrum clipping), high level clipping
  4. Tesselation (convert cube in face, sphere in faces, ...)
  5. Environement processing : lights computation (for each visible point, depend on the average normal of the point)
  6. Low level clipping (frustrum face clipping)
  7. Faces, plots, lines sorting
  8. Projecting (points and colors)
  9. Rasterinzation (effective drawing) in the image (use a clut for lights)
